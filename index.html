Aufbau und HTML Tags

Document Strukur

In einer HTML-Datei schreibt der Ersteller der Webseite den Text einer Website. Mit der HTML-Datei lässt sich ein Text strukturieren und formatieren (z.B. Schriftgröße und Schriftart). Außerdem binden Sie via HTML Links auf andere Webseiten ein oder ergänzen Ihren Text um Bilder, Videos uvm.

Tag Struktur

Tags mit einem Anfang und einem Ende sind Container oder Behälter für Informationen und strukturieren den Inhalt. Meist besteht die Beschriftung nur aus wenigen Buchstaben und ist eine Abkürzung für einen englischen Begriff.

HTML Tags — Eine Übersicht

Headline Tags

h1 — h6

Headline Tags zeichnen die Überschriften aus und sind hierarchisch geordnet.

Paragraph Tags

p

Paragraphen zeichnen Textabschnitte aus.

Link Tags

a

a Tags signalisieren dem Browser, dass ein Link eingebunden wird. Ein Link braucht immer das "Ziel Attribut" href="" damit der Browser weiß, wohin er verlinken soll.

Bild Tags

img

Bilder werden mit dem img Tag dargestellt. Bilder benötigen gleich 2 Attribute. Die Quelle: src="" und alt"".

Listen

ol / ul und li

Listen sind verschachtelte Elemente. ol steht für ordered list also eine geordnete Liste und ul für die unordered list also eine ungeordnete Liste. Der Unterschied ist in der Darstellung der einzelnen li tags, also list-items. Diese werden entweder mit Ziffern oder mit Bullets dargestellt.

Container

div

Das div-Element ist dazu gedacht, mehrere Elemente wie Text, Grafiken, Listen usw., in einen gemeinsamen Bereich einzuschließen. Es ist dazu gedacht, Bereiche zu erzeugen, die mit Hilfe von CSS formatiert werden können. div bedeutet division, also Bereich oder Abteilung.

Body Struktur

Verschachteln

Es ist gängige Praxis unterschiedliche HTML Elemente ineinander zu verschachteln. Dabei ist zu beachten: Das zuerst geöffnete Tag muss als letztes geschlossen werden.

HTML5 Elemente

HTML5 brachte einige Vorteile. Darunter die Tags header, footer, nav, main, article und section. Diese dienen der besseren Strukturierung der Seite und hilft Google die Seite besser auszulesen.

HTML5 Tags

Header

< header >

Die meisten Webseiten beinhalten am Beginn der Seite eine Navigationsleiste mit Links, Logo etc. Diese wird mit dem header tag ausgezeichnet.

Footer

< footer >

Paragraphen zeichnen Textabschnitte aus.

Navigation

< nav >

Die Haupt-Navigation wird durch ein nav-Element umschlossen, das aber auch für Unter-Navigationen verwendet werden kann.

Hauptinhalt der Seite

< main >

Der Hauptinhalt der Webseite wird im main tag abgebildet.

Artikel / Beiträge

< article >

Ein in sich geschlossener Artikel, der Überschrift (und evtl. einige sections, einen eigenen header und footer) enthält.

Kapitel, Module, Inhaltsabschnitte

< section >

Eine Section ist im Gegensatz zum Artikel ein Kapitel oder ein Abschnitt mit Inhalten. Es kann mehrere ähnliche Blöcke geben.

Why should designers code?

Mehr Möglichkeiten.

Mehr Möglichkeiten.

Viele Designer sehen Code als Einschränkung ihrer Arbeit. Das Gegenteil ist aber der Fall. Das Verständnis von Code gibt dem Designer noch mehr Möglichkeiten auf kreative Art und Weise Web / App Darstellungen zu erarbeiten.

Verstehen.

Verstehen.

Nicht nur aus technischer Sicht ist es wichtig, zu verstehen wie die Dinge funktionieren. Vor allem aus ökonomischer und zeitlicher Sicht ist es von Vorteil wenn der Designer weiß welche Auswirkungen Änderungen im Design auf die Entwicklung bzw. auf das gesamte Projekt hat.

Kommunikation.

Kommunikation.

Missverständnisse zwischen Entwicklern und Designern stehen im Arbeitsalltag meist an der Tagesordnung. "Das sollte aber nicht so", "Hier hast du was vergessen…", "Kannst du nochmal kurz?" — Typische Sätze die nicht nur in der Kommunikation zwischen Kunden und Designern fallen, sondern auch zwischen Designern und Entwicklern. Mit Verständnis von Code kann der Designer mit den Entwicklern auf Augenhöhe kommunizieren.

Es macht euch zu einem/einer besseren Designer(in).

Es macht euch zu einem/einer besseren Designer(in).

Vollständigere Designs weil man von Anfang an mitbedacht hat, dass verschiedene Versionen eines Formularfeldes benötigt werden oder eine ganzheitliche Sicht auf das Projekt Webseite. Auf einmal werden Ladezeiten, SEO oder auch das skalieren von Bildern interessant. Wie groß ist die Schrift eigentlich auf einem Tablet? Wenn ich den Slider jetzt kleiner ziehe, ist das dann noch benutzbar? Durch das Verständnis von Code nimmt man auf einmal ganz andere Dinge wahr.

Projekte selbst umsetzen.

Projekte selbst umsetzen.

Kleinere freie oder bezahlte Jobs können auf einmal selbst umgesetzt werden. Die Webseite im Abschlussprojekt ist nicht nur als einfacher Screen im Bildschirm sichtbar sondern auf einmal auch auf dem Smartphone oder Rechner benutzbar.

Wie schreibt man eine Webseite?

Auszeichnungssprache HTML

Auszeichnungssprache HTML

Der Begriff "HTML" steht für "Hypertext Markup Language", auf deutsch "Hypertext-Auszeichnungssprache". Hierbei handelt es sich um das Format, in dem Webseiten geschrieben werden.

In einer HTML-Datei schreibt der Ersteller der Webseite den Text einer Website. Mit der HTML-Datei lässt sich ein Text strukturieren und formatieren (z.B. Schriftgröße und Schriftart). Außerdem binden Sie via HTML Links auf andere Webseiten ein oder ergänzen Ihren Text um Bilder, Videos uvm.

In der Regel werden HTML-Dateien nicht mehr selber geschrieben. Webseiten-Betreibern wird diese Arbeit mit einer grafischen Oberfläche abgenommen - dies geschieht zum Beispiel via Content Management Systemen wie WordPress oder Typo3.

CSS

CSS

CSS steht für Cascading Style Sheets, was übersetzt „gestufte Stilvorlagen“ bedeutet. Es handelt sich dabei um eine Gestaltungs- und Formatierungssprache, mit der das Aussehen von HTML-Dokumenten bestimmt wird. Es geht also um Design oder Stil, nicht um den Inhalt einer Webseite. Mit CSS können Anwender zum Beispiel Schriftarten, Farben, Linien, Höhen und Breiten auf einer Webseite definieren.
